# 示例：如何组织回答（供 Agent 参考，非客户机密）

## 示例 A：用户仅问「ASIL D 要做什么测试」

**推荐结构**

1. 说明 ASIL 与 **对象**（系统/软件/硬件）及 **工作产品** 相关，不能脱离项目裁剪。
2. 列出 ISO 26262 中与验证相关的 **典型工作产品**（测试规格、静态/动态测试、接口测试等）作为检查维度。
3. 明确写出：**需结合安全计划中的方法裁剪与 ASIL 对应表**。
4. 结尾给出「若你提供 Item、ASIL、软件组件清单，可细化到可执行任务列表」。

**避免**：直接给出「这就是 ASIL D 的完整测试清单」而无裁剪说明。

---

## 示例 B：用户要求「帮我写 HARA 表」

**推荐结构**

1. 请求或列出最小输入：Item 边界、运行模式、已知危害线索、目标市场法规语境。
2. 若信息不足：用 **假设** 生成**模板行**（危害、场景、事件、S/E/C、ASIL、安全目标），并标注「待客户确认列」。
3. 提醒 HARA 变更对下游 FSC、TSC 的连锁影响。

---

## 示例 C：用户混淆 26262 与 SOTIF

**推荐结构**

1. 用一两段话区分 **故障主导** vs **性能局限/场景不足主导**。
2. 建议项目上将证据链 **分库或分册** 管理，避免混在单一「安全需求」表中不分类。
