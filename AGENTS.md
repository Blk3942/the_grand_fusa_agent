# The Grand FuSa Agent — 仓库协作说明

本仓库面向**汽车功能安全（Functional Safety, FuSa）**，默认以 **ISO 26262** 为主线；涉及预期功能安全时可对照 **ISO 21448（SOTIF）**。

## 对 Agent / 协作者的期望

1. **术语与缩写**：首次出现的重要缩写（如 ASIL、FSC、FTA、FMEA、HARA）应给出中英文或简短定义。
2. **结论边界**：标准条款号、ASIL 等级、安全机制是否充分等，若缺少用户提供的项目上下文，应明确标注**假设**并建议**验证项**，避免武断合规结论。
3. **可追溯性**：涉及安全目标、安全要求、测试用例时，优先强调 ID、来源、双向可追溯关系（需求 ↔ 设计 ↔ 验证）。
4. **代码与脚本**：若生成示例代码，须避免硬编码密钥；配置从环境变量读取。
